# 워크숍 질문 뽑기 게임 — 설계 문서

**날짜:** 2026-04-21  
**상태:** 확정

---

## 개요

직원들이 익명으로 질문을 제출하고, 피플팀이 행사 당일 재미있는 애니메이션으로 질문을 뽑아 대표가 답변하는 워크숍용 웹앱.

---

## 사용자 역할

| 역할 | 접근 방식 | 할 수 있는 것 |
|------|-----------|--------------|
| 직원 (~30명) | 기본 URL | 익명 질문 제출 (여러 개 가능) |
| 피플팀 | 같은 URL + 비밀번호 입력 | 질문 조회·수정·삭제, 뽑기 시작·제어 |
| 대표 + 전체 | 뽑기 화면 공유 | 뽑힌 질문 함께 확인 |

대표는 행사 전 질문 내용을 알 수 없음. 피플팀만 사전 열람 가능.

---

## 운영 흐름

1. **수집 단계** — 행사 전, 직원들이 URL로 접속해 질문 제출
2. **검토 단계** — 피플팀이 관리자 모드에서 질문 확인·정리
3. **행사 당일** — 피플팀이 뽑기 시작, 전체 화면 공유, 대표 답변

---

## 화면 구성

### 1. 질문 제출 화면 (기본)
- 텍스트 영역 (최대 200자)
- 글자 수 카운터
- "질문 남기기" 버튼
- 제출 완료 시 확인 메시지 표시
- 동일 사용자 복수 제출 허용 (익명이므로 식별 불필요)

### 2. 관리자 화면
- 비밀번호 입력 → 관리 모드 진입 (같은 페이지 내 모드 전환)
- 질문 목록: 제출 시각, 내용 표시
- 각 질문에 수정 / 삭제 버튼
- 이미 뽑힌 질문은 흐리게 표시
- "뽑기 시작" 버튼 → 뽑기 모드 선택 화면으로 이동

### 3. 뽑기 화면
뽑기 시작 전 4가지 모드 중 선택:

| 모드 | 설명 |
|------|------|
| 🎰 슬롯머신 | 질문들이 빠르게 스크롤되다 서서히 멈춤 |
| 🎡 룰렛 | 원판 회전 후 포인터에 멈춤 |
| 🃏 카드 뒤집기 | 카드 선택 → 플립 애니메이션으로 공개 |
| 🎲 뽑기통 | 버튼 누르거나 폰 흔들면 제비가 튀어오름 |

### 4. 질문 공개 화면
- 뽑힌 질문 텍스트 크게 표시
- 진행 표시 (예: 3 / 12)
- 전체화면 버튼 (프로젝터용)
- "다음 뽑기" 버튼 (같은 모드로 계속)
- "모드 변경" 버튼
- 남은 질문 수 표시

## 디자인 시스템

**컨셉:** 2D 픽셀 / 고전 게임 (레트로 아케이드) 스타일

| 항목 | 값 |
|------|-----|
| 폰트 | Press Start 2P (Google Fonts) |
| 배경 | `#0a0a0f` (거의 검정) |
| 메인 컬러 | `#6366f1` (인디고) |
| 액센트 | `#4ade80` (그린), `#eab308` (옐로우) |
| 버튼 | 픽셀 box-shadow로 입체감 표현, 클릭 시 눌리는 효과 |
| 입력창 | inset shadow로 CRT 모니터 느낌 |
| 애니메이션 | step-end 기반 (부드러운 전환 대신 프레임 점프) |
| 텍스트 깜빡임 | `animation: blink 1s step-end infinite` |

## Apps Script API

| 엔드포인트 (action) | 메서드 | 설명 |
|---------------------|--------|------|
| `submitQuestion` | POST | 질문 제출 |
| `getQuestions` | GET | 전체 질문 목록 (관리자용, 비밀번호 필요) |
| `deleteQuestion` | POST | 질문 삭제 (비밀번호 필요) |
| `updateQuestion` | POST | 질문 수정 (비밀번호 필요) |
| `drawQuestion` | POST | 무작위 미뽑힌 질문 1개 반환 + drawn 처리 (비밀번호 필요) |

비밀번호는 Apps Script 스크립트 속성(Script Properties)에 저장. 코드에 하드코딩 금지.

---

## 보안

- 관리자 비밀번호: Apps Script Script Properties에 저장
- 프론트에서 비밀번호를 모든 관리자 API 요청에 포함
- Apps Script에서 비밀번호 검증 후 응답
- 질문 제출은 인증 불필요 (익명)
- 스팸 방지: 클라이언트 측 제출 버튼 쿨다운 (5초)

---

## 제약 및 한계

- Apps Script cold start 1~2초 지연 가능 (30명 규모에서는 무시 가능)
- Apps Script 동시 실행 제한: 30명 동시 제출 시 일부 지연 가능 → 수집 단계는 시간을 두고 제출하므로 실질적 문제 없음
- 실시간 동기화 없음: 관리자 화면 새로고침 필요
- 모바일 흔들기(DeviceMotion API): iOS는 사용자 권한 요청 필요

